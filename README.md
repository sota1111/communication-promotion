# なごみキャラ — コミュニケーション心理的安全性低減アプリ (MVP)

子どもキャラクターの存在で場の緊張をゆるめ、気軽に話せる空気をつくるためのプロトタイプです。
会話内容は理解・解析せず、**場の「雰囲気」だけ**を変えることを目的とします。

## できること (MVP)

- マイクの音量(dB)だけをリアルタイムで検出する（会話内容は解析しない）
- 3つの状態を判定する：**静か / 普通 / 賑やか**
- 状態に応じてキャラクターが**言葉を発さず**、表情とモーションのみで反応する
  - 静か → 首をかしげる 🤔
  - 普通 → ニコニコする 🙂
  - 賑やか → ぴょんぴょん跳ねる 🎉
- 状態が変化した時のみ反応。連続反応を防ぐため**3秒のクールダウン**を設ける

## 判定ロジック

1. マイクから時間領域波形を取得し、RMS → dB に変換
2. `1秒ごと`にサンプリングし、`過去3秒`の移動平均をベースラインとする
3. 現在音量とベースラインを比較して状態を決定（`classify.js`）
   - `静か`：音量が閾値(既定 -55dB)以下
   - `賑やか`：絶対値が大きい(既定 -22dB以上) or ベースラインから +8dB 以上の急上昇
   - `普通`：上記以外
4. 状態変化かつクールダウン経過時のみキャラクターを反応させる

閾値は `classify.js` の `PARAMS` で調整できます（表情・モーションは `styles.css`）。

## プライバシー

- 音声データは**保存しません**
- 会話内容は**解析しません**
- すべて**端末内（ブラウザ内）で処理**が完結し、サーバー送信はありません

## 使い方

ブラウザはマイク利用に安全なコンテキスト（`https://` または `localhost`）を要求します。

```bash
# 静的ファイルをローカル配信して開く
npm run serve
# → ブラウザで http://localhost:8000 を開き「はじめる」→ マイクを許可
```

`index.html` をファイルとして直接開くとブラウザによってはマイクが使えないため、
上記のように `localhost` 経由で開いてください。

## テスト

判定ロジック（純粋関数）の単体テスト：

```bash
npm test   # node --test
```

## 構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | 画面（中央にキャラクター常時表示・ミニマル背景） |
| `styles.css` | キャラクターの表情・モーション（3〜5種のアニメーション） |
| `classify.js` | 場の勢い判定ロジック（単一の真実源・テスト対象） |
| `app.js` | マイク取得・dB計測・サンプリング・状態遷移・クールダウン制御 |
| `test/classify.test.mjs` | 判定ロジックの単体テスト |

## 非機能（MVP目標）

- 音量検出の遅延：最大200ms（平滑化した瞬時値を ~60fps で更新）
- アニメーション切替：100ms以内（状態クラスの切替で即時反映）
- バッテリー配慮：軽量なFFT窓・最小限の描画更新
